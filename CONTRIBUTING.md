# Contributing to Audit Agent
We welcome contributions! For security-critical changes, please first open an RFC in the docs/rfcs/ directory with full security impact analysis. All contributions must pass the security-hardened CI pipeline.

1. Fork repository
2. Create RFC for non-trivial changes
3. Implement with 100% test coverage
4. Submit PR with threat model analysis
5. Pass security review (including fuzz testing)

For security disclosures, please email security@audit-agent.org
